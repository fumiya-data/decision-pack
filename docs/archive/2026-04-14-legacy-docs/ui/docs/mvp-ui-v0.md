## 配置意図

1. 上段: 入力と読込操作（最重要導線）
2. 中段: KPI カード（意思決定に直結）
3. 下段: アラートと成果物リンク（次の行動につながる）

## 最小機能

1. JSONパス入力 + Load
2. JSON読み込み後に `scenario`, `kpi`, `alerts` を表示
3. 成果物（PNG/TXT）のパス表示またはオープン操作
4. エラー時に状態欄へ理由を表示する（ファイルなし / JSON 不正）

## 実装優先順

1. 固定レイアウト表示（ダミーデータ）
2. Load でJSON反映
3. エラーハンドリング
4. 画像/TXTオープンボタン連携
