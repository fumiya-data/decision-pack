# 推薦評価仕様

## 1. 文書の目的

この文書は、`purchase-insights` の推薦結果をどのように評価するかを定義する。

## 2. 評価対象

- 顧客ごとの次回購入候補 Top-N
- 人気順ベースライン

## 3. 初期評価方式

初期評価方式は `leave-last-order-out` とする。

- 各顧客の時系列注文列を並べる
- 最後の注文を holdout とする
- それ以前の注文を学習用履歴とする
- holdout 注文に含まれる商品集合を正解集合とする

## 4. 評価対象顧客

- 顧客ごとの注文数が `2件以上` あること
- holdout 注文に少なくとも 1 商品以上含まれること

## 5. 指標

### 5.1 主要指標

- `Hit@3`
- `Hit@5`
- `Recall@5`
- `Recall@10`
- `NDCG@5`
- `NDCG@10`

### 5.2 比較対象

- `purchase-insights` のモデル
- 全体人気順ベースライン

## 6. 指標の意味

- `Hit@K`
  - Top-K に正解商品が 1 件でも入った顧客の比率
- `Recall@K`
  - 正解集合のうち Top-K で拾えた割合
- `NDCG@K`
  - 順位を考慮した適合度

## 7. 解釈ルール

- `Hit@K` だけで良し悪しを決めない
- 人気順との差が出ているかを必ず見る
- `Recall` と `NDCG` が両方とも改善しているかを重視する

## 8. 現時点の到達条件

MVP 段階では、次を最低条件とする。

- オフライン評価が実行できること
- 上記 6 指標を出力できること
- 人気順ベースラインとの差分を表示できること
- 評価対象顧客数を同時に出力できること

## 9. 非要求

- 初期段階ではオンライン A/B テストは行わない
- 初期段階では機械学習モデルの自動学習基盤は必須としない

## 10. 将来拡張

将来的には次を検討する。

- 期間固定の train / validation / test 分割
- セグメント別評価
- 商品カテゴリ別評価
- 売上金額重み付き評価
