# 機能要件

## 1. 顧客データ前処理

- 顧客CSVを受け取り、正規化済み顧客データを生成する
- 整形失敗や欠損を issue として記録する
- PostgreSQL へ顧客情報を永続化できる

## 2. 商品・購入履歴・在庫

- 商品品目リストを管理できる
- 受注と受注明細を取り込み、顧客が何を買ったかを追跡できる
- 在庫残高を品目単位で持てる
- 在庫一覧の絞り込み検索ができる

## 3. 購入傾向分析

- 顧客ごとの購入履歴を集約できる
- 顧客ごとの次回購入候補 Top-N を算出できる
- 品目ごとの将来需要を集約できる

## 4. 意思決定支援

- 品目別需要予測を入力にして在庫リスクを計算できる
- 補充量候補を算出できる
- 将来の資金影響を評価できる
- シナリオ単位で比較できる

## 5. GUI

- 顧客一覧を表示できる
- 顧客詳細画面で購入履歴を見られる
- 顧客ごとの次回購入候補を表示できる
- 在庫一覧と在庫リスク一覧を表示できる
- 品目検索と絞り込みができる
- シミュレーション実行結果を見られる
- シミュレーションのレポート成果物を表示できる

## 6. API

- 顧客、購入履歴、推薦結果、在庫、シミュレーション結果、レポート成果物を取得できる
- ETL、分析、シミュレーションをジョブとして起動できる
- 長時間処理の状態を追跡できる
