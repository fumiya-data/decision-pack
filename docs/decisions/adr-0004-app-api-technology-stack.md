# ADR-0004 `app-api` の実装技術

## 状態

採用済み

## 背景

`app-api` は GUI の唯一の境界であり、業務データ参照とジョブ起動の両方を扱う。
このプロジェクトは Rust を中心に構成しており、型安全性と実行時依存の単純さを維持したい。
将来の AWS 配置でも、ローカルと本番で実装を分けない形が望ましい。

## 決定事項

- `app-api` は Rust で実装する
- Web フレームワークは Axum を使う
- PostgreSQL アクセスは SQLx を使う
- ORM は採用しない
- DB 操作は原則として手書き SQL で行う

## 影響

- GUI 境界と既存 Rust モジュールの言語一貫性が保たれる
- SQL を明示的に管理しやすくなる
- DB の意味が ORM モデルへ吸収されにくくなる
- `Lambda` と `ECS` のどちらにも載せやすい API 実装になる
