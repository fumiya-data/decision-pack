# ADR-0007 複数データソース統合取込の基本方式

## 状態

採用済み

## 背景

今は CSV が中心だが、将来は複数の業務データソースを取り込む可能性がある。
ソースごとに最終テーブルへ直接書き込むと、正規化責務とソース依存の差異が混ざりやすい。
統合取込の土台を早めに固定しておきたい。

## 決定事項

- 入力ソースごとに adapter を用意する
- adapter の出力は共通 staging へ集約する
- 正規化テーブルへの反映は staging から行う
- ソースごとに最終テーブルへ直接書き込む方式は採らない

## 影響

- ソース追加時の差分が adapter に閉じやすくなる
- 正規化処理とソース依存処理を分離できる
- 監査と再取込の設計がしやすくなる
