# ADR-0006 MOQ と lot size の適用位置

## 状態

採用済み

## 背景

MOQ や lot size は需要そのものではなく、補充判断の制約である。
これを ETL や需要予測の段階で織り込むと、予測値と補充制約が混ざり、責務が曖昧になる。
補充提案の説明責任を保つため、制約の適用位置を明確にしたい。

## 決定事項

- MOQ と lot size は `decision-engine` の補充提案ロジックで適用する
- `customers-etl`、`commerce-etl`、`purchase-insights` は制約を織り込んだ需要値を作らない
- 需要予測と補充制約は別の入力として扱う

## 影響

- 需要予測と補充判断の責務が分離される
- 補充提案の根拠を説明しやすくなる
- 補充制約の変更が分析ロジックへ波及しにくくなる
