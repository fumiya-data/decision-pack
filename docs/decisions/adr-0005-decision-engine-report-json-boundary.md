# ADR-0005 `decision-engine` レポート JSON 境界

## 状態

採用済み

## 背景

`desktop-ui` と `reporting` は `decision-engine` の内部実装ではなく、レポート JSON 契約に依存させたい。
一方で、要約だけでは GUI が必要とする在庫リスクや品目別結果を表示しきれない。
内部トレースをすべて出すと契約が重くなり、互換性維持も難しくなる。

## 決定事項

- レポート JSON は `simulation_report_v0.1` をベースに互換性を保って拡張する
- 拡張範囲は GUI 表示に必要な実行要約、品目別結果、アラート、成果物参照までとする
- 日次トレースや内部中間値は既定では含めず、必要時のみ追加拡張で扱う

## 影響

- `desktop-ui` はレポート成果物を API 経由で安定して表示できる
- `reporting` は内部実装に依存せず JSON 契約だけを見ればよい
- 契約が過剰に肥大化するのを防げる
