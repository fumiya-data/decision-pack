# ADR-0001 命名規則と文書配置

## 状態

採用済み

## 背景

既存文書は配置が散乱し、ファイル名の規則が統一されていなかった。
今後は仕様先行でトップダウン式にプロジェクトを進行させることに決定している。
仕様先行で進めるための文書形式を先に統一する必要があり、正本配置と命名規則を先に固定することとした。

## 決定事項

- 文書ファイル名と文書ディレクトリ名は英語の `kebab-case` に統一する
- 本文は日本語を正本とする
- `docs/` 配下を現行文書の正本とする
- `docs/progress/` の進捗記録は `yyyy-mm-dd-hhmm-<summary>.md` 形式とし、更新単位でファイルを分ける
- 旧文書は `docs/archive/` に隔離する
- コードファイルは各言語の標準規約に従う

## 影響

- 新規文書はすべて `docs/` に配置される。
- ルートやサブディレクトリに文書が散乱するのを防ぐ。
