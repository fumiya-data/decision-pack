## 実施内容

- `db/migrations/202604172120_initial_schema.sql` を追加し、顧客、商品、受注、在庫、需要予測、シミュレーション結果の永続化テーブルを定義した。
- `customers-etl` に PostgreSQL 永続化を追加し、整形済み顧客データと整形時問題ログを DB へ書き込めるようにした。
- `commerce-etl` を実装し、商品 CSV、受注 CSV、受注明細 CSV、在庫 CSV を PostgreSQL へ投入できるようにした。
- `purchase-insights` を実装し、購入履歴から次回購入候補と品目需要予測を算出して DB へ書き込めるようにした。
- `decision-engine` にシナリオ実行入口を追加し、在庫・資金・補充判断をまとめてシミュレーションできるようにした。
- `app-api` を実装し、顧客、購入履歴、次回購入候補、在庫、リスク、シミュレーション一覧、シミュレーション詳細、レポート JSON を HTTP API で取得できるようにした。
- `app-api` にシミュレーション起動 API を追加し、需要予測と在庫残高をもとに `decision-engine` を実行し、結果を DB に保存できるようにした。
- `desktop-ui` を API 前提の薄い GUI へ切り替え、顧客タブ、在庫タブ、シミュレーションタブから `app-api` を通じて情報を取得・表示できるようにした。
